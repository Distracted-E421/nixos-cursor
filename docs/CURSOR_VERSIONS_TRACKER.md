## Our Strategy

### For Custom Modes
Since Cursor removed custom modes in 2.1.x, we implement **external custom modes** via:

1. **Cursor Studio Modes Module** (`cursor-studio-egui/src/modes/`)
   - Store mode definitions in `~/.config/cursor-studio/modes/`
   - Generate `.cursorrules` files from mode definitions
   - Inject into `.ai-workspace/` for context awareness

2. **Mode Injection Targets**
   - `.cursorrules` - Main project rules file
   - `.cursor/rules/*.mdc` - Per-rule files
   - `.ai-workspace/hints.md` - AI workspace hints

3. **Built-in Modes**
   - `Agent` - Full autonomous with all tools
   - `Code Review` - Read-only analysis
   - `Maxim` - Your Obsidian-specific config
   - `Planning` - Plan before acting

### Version Recommendation

| Use Case | Recommended Version |
|----------|---------------------|
| **Full custom modes** | 2.0.77 (last with native support) |
| **Latest features + external modes** | 2.2.27 + Cursor Studio modes |
| **Stable testing** | 2.0.64 or 2.0.43 (verified) |

## URL Pattern

All URLs follow this pattern:
```
https://downloads.cursor.com/production/{commit_hash}/linux/{arch}/Cursor-{version}-{arch_suffix}.AppImage
```

Where:
- `{commit_hash}` - Git commit or build hash
- `{arch}` - `x64` or `arm64`
- `{arch_suffix}` - `x86_64` or `aarch64`
